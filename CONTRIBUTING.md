# Contributing

Found a bug? Have an improvement? Contributions welcome!

## How to Contribute

1. **Fork this repo**
2. **Create a feature branch:** `git checkout -b feature/my-improvement`
3. **Make your changes**
4. **Test the skill** (run a few test cases to verify it still works)
5. **Submit a PR** with a clear description of what changed and why

## What We're Looking For

- **Bug fixes**: Edge cases, parsing issues, output format problems
- **Improvements**: Better error handling, performance, reliability
- **New skills**: Roadmap cascade, competitive brief, etc. (see README for planned skills)
- **Documentation**: Clearer examples, better troubleshooting guides

## Skill Development Guidelines

- Keep the SKILL.md under 500 lines
- Use the existing script structure (scripts/, references/, examples/)
- Test with realistic inputs before submitting PR
- Update the main README if adding a new skill
